# 00 · 개요 (Overview)

## 비전
밥 먹고 **그 자리에서 사진 한 장 찍으면 끝**. AI가 칼로리·탄단지를 추정해 기록하고,
시간이 지나면 "일주일/한 달간 이만큼 먹었구나", "체중이 이렇게 변했구나"를
한눈에 보고, 보고서와 가벼운 조언까지 받는 **개인용 기록 도구**.

## 핵심 가치
- **기록 마찰 최소화** — 찍는 데 3초. 그게 전부.
- **조회의 즐거움** — 히스토리·차트·보고서를 보는 맛이 있어야 계속 쓴다.
- **계속 붙여 나가는 도구** — 완성품이 아니라, 평생 만지작거리며 기능을 얹는 개인 프로젝트.

## 주요 사용 시나리오
1. 점심을 먹고 음식 사진을 찍는다 → AI가 "김치찌개, 공기밥 / 약 650kcal, 단백질 28g…" 추정 → 숫자 살짝 보정 → 저장.
2. 저녁에 오늘 하루 섭취량을 차트로 확인.
3. 주말에 "이번 주 보고서"를 눌러 주간 섭취 패턴 + 체중 추세 + AI 조언을 읽는다.
4. 가끔 체중을 입력해 추세선을 갱신.
5. (Phase 4) 점심·저녁 시간에 "식단 기록하셨어요?" 푸시를 받는다. 단, 이미 기록했으면 안 옴.

## 목표 (Goals)
- 사진 → 칼로리/탄단지 추정 → 보정 → 저장의 빠른 루프
- 일/주/월 단위 섭취량 조회 (차트)
- 체중 기록 + 추세 차트
- 주간/월간 보고서 + AI 조언
- 안드로이드 홈 화면에 앱처럼 설치 (PWA)
- 정해진 시간 푸시 알림 (스마트 스킵 포함)

## 비목표 (Non-Goals) — 명시적으로 안 함
- ❌ **판매/수익화** — 무료. 결제·구독 없음. (단, 지인 몇 명이 각자 무료로 쓰는 멀티유저는 함 — D16.)
- ❌ **iOS 대응** — 안드로이드만. iOS 전용 PWA 푸시 제약 신경 안 씀.
- ❌ **공유/팀/권한 기능** — 멀티유저지만(D16) 각자 독립 개인 공간. 다른 사람 기록을 보거나 공유·협업하는 기능은 없음.
- ❌ **정밀 영양 측정** — AI 추정치는 어디까지나 근사값. "참고용"이 기본 포지션.
- ❌ **앱스토어 출시** — 필요해지면 나중에 Capacitor로 감싸는 길은 열어두되, 지금은 PWA로 충분.
- ❌ **소셜/공유 기능** — 혼자 보는 기록.

## 제약 (Constraints)
- 플랫폼: 안드로이드 모바일 브라우저(크롬) + 홈 화면 설치 PWA
- 사용자: 소수 (본인 + 초대한 지인 몇 명), 각자 독립 데이터 (D16)
- 인프라: Neon(Postgres) + Vercel — 이미 익숙한 스택 재사용
- 정확도: AI 비전 추정은 부정확할 수 있음 → 저장 전 수동 보정 화면 필수
