# 05 · 기능 명세 (Features)

> 화면 단위로 정리. 하단 탭 네비게이션 4개 + 보조 화면.
> 우선순위는 `09-roadmap.md`의 Phase로 관리.

## 하단 탭 네비게이션
`기록` · `히스토리` · `체중` · `보고서` — 폰 하단 고정, 앱 느낌의 핵심.

---

## 1. 기록 (홈) — `/`
**목적:** 최소 마찰로 끼니를 남기고, 오늘 하루를 한눈에 본다 — **기록 기능 + 홈 대시보드를 한 화면에서 겸한다**(2026-06-24 확정. 탭 라벨은 "기록" 유지 — 핵심가치가 "찍는 데 3초"라 라벨도 행동 중심으로 둠, `00`).
- **오늘 누적 대시보드** (참고 앱 스샷 기반): 섭취 kcal/목표 kcal·매크로(탄/단/지) g 진행률은 **확정(D20, Phase 7)** — `/settings`에서 신체정보·목표를 입력해야 보임, 미입력 시 "설정에서 목표를 입력하면 보여요" 안내. 평가 문구·경고색 없이 숫자/바만(D15 칼로리 강박 방지). 순탄수·당류·나트륨 누적, 운동 기록 반영, 보상(딸기) 누적은 Phase 8+ 백로그. 계산 공식은 `03` "계산값" 참고.
  - **주의(마찰 최소화 원칙 유지, `00`):** 대시보드가 늘어나도 "사진 찍기" 버튼은 스크롤 없이 항상 바로 보여야 함. 참고 앱의 "간단/상세/한눈에" 토글처럼 정보 밀도를 사용자가 줄일 수 있게 하는 것도 백로그 후보.
- 큰 "사진 찍기 / 갤러리" 버튼. 카메라 즉시 호출(`<input type="file" accept="image/*" capture="environment">`).
- 사진 선택 → 로딩 → AI 추정 결과 카드 표시:
  - 인식된 음식 항목 리스트
  - kcal / 단백질 / 탄수 / 지방
- **보정 영역(필수):** 각 숫자를 사용자가 즉석 수정 가능. 항목 추가/삭제도.
- 메모 입력(선택). **먹은 날짜·시간은 현재 항상 지금(now)으로 저장됨 — 선택 UI 없음(백로그, `09` Phase 7+ "지난 날짜 기록 입력" 참고, 2026-06-25 확인). API(`/api/meals` POST)는 `eatenAt`을 받을 준비가 되어 있어 UI만 추가하면 됨.**
- **수동 입력(검색, D18)**: 사진 없이 음식 이름으로 검색해 추가. 식약처 DB 검색 결과가 없으면 AI 텍스트 추정으로 폴백, 같은 보정 화면 재사용. **(Phase 6)** 검색창이 비어 있으면 즐겨찾기 + 자주 먹는 음식(`usage_count` 순)을 먼저 보여줘 검색 없이 한 탭으로 재입력. 모든 음식 항목에 별(☆/★) 토글로 즐겨찾기 지정. 검색 결과가 없으면 AI 추정 버튼 옆에 "직접 등록"(이름+기준량+영양정보) 버튼.
- "저장" → meals insert → 토스트 후 홈 리셋.
- (백로그, 낮은 우선순위, 2026-06-24 확정) **운동 퀵 입력**: `/weight` 체중 입력과 같은 패턴으로 운동 시간을 기록 → 소모 칼로리 계산해 대시보드에 반영. `03`의 `exercise_logs` 참고.

## 2. 히스토리 — `/history`
**목적:** "얼마나 먹었구나"를 본다.
- 기간 토글: **일 / 주 / 월**.
- 일: 그날의 기록을 **끼니(아침/점심/저녁/간식)별로 그룹핑**해서 보여줌. 같은 끼니를 여러 기록으로 나눠 입력해도(예: 점심을 두 번에 나눠 입력) 그 끼니 그룹은 하나로 합쳐 보여줌 — 그룹 합계 kcal + 그룹에 속한 모든 기록의 음식 항목을 이름/양/단위/kcal별로 나열(사진 보관 시 기록별 썸네일 포함, 탭하면 원본 비율로 크게 보기).
- 주/월: 날짜별 총 kcal 막대 차트(Recharts) + 평균/합계. 탄단지 누적도 옵션.
- 끼니 그룹 안의 개별 기록 단위로 **수정**/**삭제** 가능(보정 화면 재사용 + 삭제 확인 다이얼로그). 나트륨/당류/식이섬유 등 확장 영양정보는 범위 밖(Phase 7+ 백로그).

## 3. 체중 — `/weight`
**목적:** "체중이 이렇게 변했구나"를 본다.
- 빠른 입력: 숫자 패드로 체중(kg), 선택적 체지방률.
- 추세 라인 차트(기간 토글). 목표 체중선(점선, `/settings`에 `target_weight_kg` 입력 시) — **확정(D20, Phase 7)**.
- 최근 기록 리스트(수정/삭제).

## 4. 보고서 — `/report`
**목적:** 패턴 파악 + 가벼운 조언.
- 기간 선택: **주 / 월**.
- "보고서 생성" → 해당 기간 집계 + 체중 추세를 AI에 넘겨 자연어 요약:
  - 평균 섭취 칼로리/매크로, 끼니 패턴, 눈에 띄는 점
  - 체중 변화와의 연결
  - **균형 잡힌, 비강압적인 조언** (극단적 제한 권하지 않음 — `06` 프롬프트 가이드 준수)
- 생성 결과는 화면에 카드로(통계 카드 + "밥로그의 한마디" 카드). **캐싱 안 함** — 누를 때마다 새로 생성(`09` Phase 3 결정).

## 5. 설정 — `/settings` (D20, Phase 7)
**목적:** 목표 kcal/매크로 계산에 쓸 1회성 신체정보·목표를 입력한다. 바텀탭에 없음 — 4개 메인 탭 공용 `AppBar`(`components/AppBar.tsx`)의 ⚙ 드롭다운("설정"/"로그아웃")으로 어느 탭에서든 진입.
- 입력 항목: 생년, 키(cm), 성별, 활동량(5단계), 목표(감량/유지/증량), 목표 체중(kg), 주간 변화율(kg/주)
- 저장 → `/api/profile`(GET/POST) → `user_profiles` upsert. 계산(BMR/TDEE/목표kcal/매크로g/BMI)은 `lib/health/goals.ts`에서 결정적으로 파생(LLM 아님).
- 미입력 상태도 정상 — 기록 홈 대시보드·체중 차트는 목표 관련 표시만 숨기고 나머지는 그대로 동작.
- **앱바/뒤로가기/하단탭 일관성에 미해결 이슈 있음 — `09` Phase 8+ "앱바/설정 화면 UX 다듬기" 참고.**

## 보조/후순위 기능 (계속 붙일 것들)
- 바코드(Open Food Facts)로 가공식품 정확도 보정
- 푸시 알림 (Phase 4, `08`)
- 데이터 내보내기(JSON/CSV)
- 음식별 영양정보 확장(나트륨/당류/식이섬유 등) — `foods`/식약처 매핑 활용 (`03`, `06` 참고)
- 보상 시스템(기록할 때마다 "딸기" 적립) — 사용처 미정, 인아웃 사과 참고. 적립 로직(`reward_ledger`, `03`)은 선제 준비. D14(인아웃 게이미피케이션 비목표)와 관계 재검토 필요 (`01` 미해결 참고)
- (검토, 낮은 우선순위) 운동/활동 기록(운동시간→소모kcal) — 참고 앱엔 있으나 범위가 큼(MET 계산 등), 도입 여부 미정 (`01` 미해결 참고)

> 새 기능을 붙일 때: 이 문서에 화면/동작을 추가하고, `09-roadmap`에 작업으로 넣고, 끝나면 `PROGRESS`에 기록.
